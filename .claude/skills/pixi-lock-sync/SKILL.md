# /pixi-lock-sync — Lock File Synchronization

Ensure `pixi.lock` stays in sync with `pyproject.toml` after dependency changes.

## When to Use

Run this skill after modifying any dependency-related sections in `pyproject.toml`:
- `[project.dependencies]`
- `[project.optional-dependencies]`
- `[tool.pixi.dependencies]`
- `[tool.pixi.feature.*.dependencies]`

## Steps

1. **Check for pyproject.toml changes** by running:
   ```bash
   git diff --name-only HEAD -- pyproject.toml
   ```
   If no changes, also check unstaged:
   ```bash
   git diff --name-only -- pyproject.toml
   ```

2. **If pyproject.toml was modified**, run:
   ```bash
   pixi install
   ```
   This regenerates `pixi.lock` from the updated dependencies.

3. **Verify the lock file was updated**:
   ```bash
   git diff --stat -- pixi.lock
   ```

4. **Stage both files together**:
   ```bash
   git add pyproject.toml pixi.lock
   ```

5. **Report** what changed: which dependencies were added/removed/updated and confirm the lock file is in sync.

## Rules

- ALWAYS run `pixi install` after any `pyproject.toml` dependency change
- ALWAYS stage `pixi.lock` alongside `pyproject.toml` in the same commit
- NEVER edit `pixi.lock` directly — it is generated by `pixi install`
- CI uses `--locked` and will fail if the lock file is stale
